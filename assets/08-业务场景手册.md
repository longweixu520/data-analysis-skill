# 08 - 业务场景手册

按常见交付类型给出**默认分析路径**，可直接写入 Data Contract。

---

## 场景 A：城市 / 区域横向对标

**典型数据**：统计年鉴、区域经济表、多城市多年指标

**核心问题**：某年各城市水平如何？多年趋势？综合排名？

**默认路径**：
1. 侦查：识别主体列、年份列、模式 A/B/E
2. 工程：长表 + 主体标准化
3. 计算：最新年排名 → 同比 → 熵值法综合指数（若多指标）
4. 交付：排名条形图 + 趋势折线 + 热力图

**默认图表**：Fig1 排名、Fig2 综合指数趋势、Fig3 权重

**常见坑**：「北京市」vs「北京」；GDP 单位亿元 vs 元

---

## 场景 B：企业经营分析

**典型数据**：年报摘录、财务指标表、多实体对比

**核心问题**：盈利能力、成长性、相对行业位置

**默认路径**：
1. 侦查：确认报告期、合并报表 vs 母公司
2. 工程：指标分类（盈利/偿债/成长/运营）
3. 计算：同比、CAGR、行业分位（若有行业数据）
4. 交付：关键指标趋势 + 雷达图（≤5 企业）

**综合评价**：仅在用户要求「综合竞争力」时启用

---

## 场景 C：行业 / 品类结构

**典型数据**：产量、份额、占比表

**核心问题**：结构变化、集中度、头部主体

**默认路径**：
1. 计算：占比、CR4/CR8（若适用）、结构变化
2. 交付：堆叠柱状图、份额趋势

**不做**：无依据的集中度预测

---

## 场景 D：问卷 / 调查汇总

**典型数据**：李克特量表、多选题频数、交叉表

**核心问题**：描述分布、组间差异

**默认路径**：
1. 工程：题项编码、反向题处理
2. 计算：频数、均值、分组对比；必要时 t 检验 / 卡方（见 06）
3. 交付：分布图、组间对比图

**长表字段**：主体=受访者或样本组，指标=题项，年份可换为「批次」

---

## 场景 E：面板数据政策评估

**典型数据**：政策前后、多地区面板

**核心问题**：政策后指标是否变化

**默认路径**：
1. 描述：政策前后均值对比（同地区）
2. 谨慎推断：双重差分需专门设计，默认只做描述对比
3. 报告：明确「相关不等于因果」

---

## 场景 F：仅数据清洗交付

**用户只要**：干净的长表，不要报告

**路径**：侦查 → 工程 → `validate_delivery.py`（不要求 --strict）

**产出**：01/02/03 CSV + `src/run_pipeline.py`
