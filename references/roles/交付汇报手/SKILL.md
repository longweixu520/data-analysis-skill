# 交付汇报手 (Delivery Reporter)

将前三阶段的**证据链**整理为业务方可读、可审计、可引用的正式交付物。

---

## 核心职责

1. **Figure Contract**：每张图证明一个独特主张
2. **中文图表**：标题、坐标轴、图例、注释全部中文
3. **报告撰写**：过程透明、结论有据、风险明示
4. **格式交付**：Markdown 报告 + 可选 Word + 图表双格式
5. **交付质检**：运行 validate_delivery，逐项勾选清单

---

## 默认图表清单

| 图表 | 适用场景 | 类型 |
|------|----------|------|
| 最新年份综合排名 | 综合评价项目 | 横向条形图 |
| 综合指数年度趋势 | 多年面板 | 折线图 |
| 权重分布 | 熵值法/AHP | 条形图或饼图 |
| 重点指标热力图 | 多指标对比 | 热力图 |
| 前几名雷达图 | 分维度对比 | 雷达图（≤5 主体） |

**防冗余**：每张图必须对应 Data Contract 中的一个子问题证据。

## 交付三原则

```
1. 先风险后结论 — 口径问题、补值假设必须前置说明
2. 图表即论证 — 去掉图后结论仍成立，则图是冗余的
3. 交付可复现 — 报告中的每个数字能在 output/ 找到来源文件
```
