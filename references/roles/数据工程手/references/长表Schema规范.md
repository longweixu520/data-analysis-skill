# 长表 Schema 规范

## 必需列

| 列名 | 类型 | 说明 |
|------|------|------|
| 来源文件 | string | 不含路径或含，全文一致即可 |
| 来源工作表 | string | Excel sheet 名；CSV 填文件名 |
| 主体 | string | 城市/企业/地区，全文统一命名 |
| 年份 | int | 四位年份；月度用 YYYYMM |
| 指标名称 | string | 与术语表一致 |
| 指标分类 | string | 如「经济」「社会」「环境」 |
| 原始值 | float | 可为 NaN |
| 清洗值 | float | 工程阶段填充 |
| 单位 | string | 与术语表一致 |
| 说明 | string | 备注、补值方法、口径变更 |
| 指标方向 | string | `正向` 或 `逆向` |
| 是否纳入综合指数 | string | `是` 或 `否` |

## 命名约定

- 主体：去掉「市」「省」后缀需全文统一，选一种策略并写入术语表
- 指标：不用原始列名中的空格和特殊字符
- 年份：禁止 `19`、`2020年` 等混用

## 类型约束

```python
DTYPE_MAP = {
    "年份": "Int64",
    "原始值": "float64",
    "清洗值": "float64",
}
```

## 唯一性

同一 `(主体, 年份, 指标名称, 来源工作表)` 不应重复。重复时保留最新或合并规则写入说明。

## 导出

```python
df.to_csv("output/02_清洗后长表.csv", index=False, encoding="utf-8-sig")
```

`utf-8-sig` 确保 Excel 打开中文不乱码。
