# 数据工程工作流程

## 输入

- `数据侦查报告.md`
- `Data_Contract.md`
- `术语与口径表.md`
- 原始数据（只读）

## 输出

- `output/01_原始标准长表.csv`
- `output/02_清洗后长表.csv`
- `output/03_数据质量检查表.csv`
- `src/parse_*.py`, `src/clean.py`, `src/run_pipeline.py`

## Step 1：搭建项目结构

```
项目/
├── 原始数据/          # 只读
├── output/
├── src/
│   ├── parse/         # 按 sheet 拆分解析器
│   ├── clean.py
│   └── run_pipeline.py
└── requirements.txt
```

`run_pipeline.py` 必须支持一键重跑：

```bash
python src/run_pipeline.py
```

## Step 3：数据质量检查

生成 `03_数据质量检查表.csv`：

| 检查项 | 主体 | 指标 | 年份 | 问题类型 | 处理建议 |
|--------|------|------|------|----------|----------|
| 缺失 | 北京 | GDP | 2020 | 结构性空值 | 保留 |
| 异常 | 某市 | 人口 | 2021 | 计数出现小数 | 置空 |

问题类型枚举：`表头错位` `年份识别错误` `主体不统一` `文本混入数值` `单位数小数` `比例超100%` `极值异常` `重复记录` `结构性空值`

---

## Step 4：清洗

在 `02_清洗后长表.csv` 中新增/更新 `清洗值` 列。

**补值优先级**（详见 `缺失与异常处理.md`）：

1. 保留原始缺失
2. 结构错位异常 → 置空
3. 同主体同指标时间序列线性插值
4. 同年中位数
5. 同指标全样本中位数
6. 仍不可靠 → 不补，报告说明

每次补值写入 `说明` 列，如 `线性插值(2019,2021)`。

---

## Step 5：工程质检

- [ ] `run_pipeline.py` 可重复运行，结果一致
- [ ] 原始值列未被覆盖
- [ ] 质量检查表覆盖所有已知问题
- [ ] 主体名称已统一（如「北京市」→「北京」）

通过后移交分析计算手。
