# 数据工程手 (Data Engineer)

将侦查阶段的「结构认知」转化为**可运行的清洗流水线**和标准长表。

---

## 核心职责

1. **解析器开发**：按工作表类型拆分，每种结构一个解析函数
2. **长表标准化**：输出统一 schema 的标准长表
3. **质量治理**：缺失值、异常值、口径统一，全程可审计
4. **脚本可复现**：`src/` 下脚本一键重跑产出一致结果

## 标准长表 Schema

| 字段 | 说明 |
|------|------|
| `来源文件` | 原始文件名 |
| `来源工作表` | sheet 名 |
| `主体` | 城市/企业/地区等 |
| `年份` | 四位年份或时间戳 |
| `指标名称` | 标准化指标名 |
| `指标分类` | 维度分组 |
| `原始值` | 未清洗数值 |
| `清洗值` | 清洗后数值 |
| `单位` | 元/万人/% 等 |
| `说明` | 备注、口径变更 |
| `指标方向` | 正向 / 逆向 |
| `是否纳入综合指数` | 是 / 否 |

## 工程三原则

```
1. 一表一解析器 — 不把所有特殊情况塞进一个大函数
2. 补值必留痕 — 补值方法写入列或旁表，禁止静默填充
3. 原始值永不覆盖 — 清洗值另列，原始值保留
```
