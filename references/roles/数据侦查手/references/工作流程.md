# 数据侦查工作流程

## 阶段目标

产出三份文档，使后续阶段**无需再猜测**数据结构与分析范围。

---

## Step 1：项目盘点（15 min）

```markdown
检查项：
- [ ] 列出目录中所有数据文件（扩展名、大小、修改时间）
- [ ] 区分「用户指定原始文件」与「历史产出/缓存」
- [ ] 是否存在已有脚本？理解后决定重用或重写
- [ ] 用户要求的交付格式（CSV / Excel / Word / 图表语言）
```

运行快速画像：

```bash
python scripts/profile_data.py 原始数据/xxx.xlsx
python scripts/profile_data.py 原始数据/xxx.csv
```

---

## Step 2：结构抽样（核心）

对每个工作表 / 文件：

1. 读取前 15 行（含可能的单位行、表头行）
2. 判断结构模式（见 `表结构识别手册.md`）
3. 记录：主体列、年份列、指标列、单位行、备注列
4. 标注「一表一模式」— 不同 sheet 可能不同模式

**禁止**：未抽样就写解析器。

---

## Step 3：口径梳理

对每个识别出的指标：

| 字段 | 记录内容 |
|------|----------|
| 指标名称 | 标准化名称 |
| 原始列名 | 表中实际列名 |
| 单位 | 元 / 万人 / % / 指数 |
| 量纲 | 总量 / 强度 / 占比 / 增速 |
| 方向 | 正向（越大越好）/ 逆向 |
| 是否百分数 | 0.23 还是 23% |

---

## Step 4：风险登记

写入 `数据侦查报告.md` 的「风险与限制」章节：

- 合并单元格导致错位
- 某年统计口径变更
- 指标覆盖年份不足
- 疑似示例数据或异常极值
- 多工作表结构不统一

---

## Step 5：建立 Data Contract

按 `Data_Contract模板.md` 填写，经以下门禁后进入工程阶段：

- [ ] 核心问题一句话清晰
- [ ] 每个子问题有对应方法
- [ ] 交付清单完整
- [ ] 风险已登记

---

## 产出模板：`数据侦查报告.md`

```markdown
# 数据侦查报告

## 1. 项目概述
## 2. 数据来源清单
## 3. 工作表结构摘要
### 3.1 [Sheet名] — 模式 [A/B/C/...]
## 4. 指标口径表
## 5. 数据质量初判
## 6. 分析计划（对应 Data Contract）
## 7. 风险与限制
```
