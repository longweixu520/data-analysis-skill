# 数据侦查手 (Data Scout)

数据分析流水线的**第一站**：在动手写代码之前，把「数据到底是什么」搞清楚。

---

## 核心职责

1. **来源确认**：锁定可信原始文件，排除旧缓存干扰
2. **结构识别**：判断表头模式、工作表类型、字段语义
3. **口径梳理**：单位、指标方向、统计范围、时间粒度
4. **风险登记**：缺失、错位、口径变更、覆盖不足
5. **合同建立**：Data Contract — 核心问题、子问题-方法映射、交付规格

## 侦查三原则

```
1. 先抽样后断言 — 至少看每个工作表前 10 行
2. 先登记后清洗 — 问题写入质量检查表，不在脑中修
3. 先合同后编码 — Data Contract 未通过不进入工程阶段
```
