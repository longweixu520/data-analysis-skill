# 场景路由表

按用户意图选择**最小必要流程**，避免四阶段全跑。

---

## 路由决策

```
用户要什么？
├─ 只看数据结构 / 修复解析
│   └─ 侦查 → 工程（跳过计算、交付）
├─ 只要清洗后的长表
│   └─ 侦查 → 工程 → 输出 CSV（跳过计算、交付）
├─ 排名 / 同比 / 单指标分析
│   └─ 侦查 → 工程 → 计算（简版报告可选）
├─ 综合指数 + 正式报告
│   └─ 全四阶段
└─ 已有长表，只要分析和报告
    └─ 跳过侦查（快速校验 schema）→ 计算 → 交付
```

---

## 场景速查

| 用户说法 | 阶段 | 重点产出 |
|----------|------|----------|
| 「帮我看看这个表什么结构」 | 侦查 | 侦查报告 |
| 「生成标准长表」 | 侦查+工程 | 01/02 CSV |
| 「做城市排名」 | 全或 工+算+交付 | 排名表 + 条形图 |
| 「综合指数 + 权重」 | 全四阶段 | 权重表 + 综合指数 |
| 「同比环比分析」 | 工+算 | 同比分析.csv |
| 「出正式中文报告」 | 全四阶段 + strict 质检 | 报告.md + 图表 |
| 「修复解析 bug」 | 工程为主 | src/ + 02 CSV |

## 轻量 Data Contract

简单任务可用三行版：

```markdown
核心问题：[一句话]
数据源：[文件路径]
交付：[列出文件名]
```

复杂任务用完整模板：`references/roles/数据侦查手/references/Data_Contract模板.md`

---

## 阶段跳过规则

| 跳过 | 条件 | 风险 |
|------|------|------|
| 侦查 | 用户确认结构已知且术语清晰 | 口径误解 |
| 工程 | 输入已是标准长表 | 需 schema 校验 |
| 计算 | 仅需清洗交付 | — |
| 交付 | 用户只要 CSV | 无图表报告 |

跳过时在回复中**明示跳过了什么**及假设。
