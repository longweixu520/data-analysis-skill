# 任务决策树

Agent 收到数据分析任务时，**先走决策树再写代码**。

```
用户任务
│
├─ 有原始 Excel/CSV？
│   ├─ 否，只有结论需求 → 要求提供原始文件
│   └─ 是 ↓
│
├─ 结构是否已知？
│   ├─ 否 → 阶段①侦查：profile_data + probe_excel + Data Contract
│   └─ 是 ↓
│
├─ 是否模式 A（主体×年份矩阵）？
│   ├─ 是 → parse_matrix.py 或 parse_from_config.py
│   └─ 否 → 按 assets/09 写自定义解析器
│
├─ 是否已有 02_清洗后长表？
│   ├─ 否 → 阶段②工程：run_pipeline.py（项目 src/）
│   └─ 是 ↓
│
├─ 要什么产出？
│   ├─ 只要 CSV → analyze_long_table.py
│   ├─ 要图表 → + plot_delivery.py
│   ├─ 要报告 → + generate_report.py + build_html_report.py
│   ├─ 要 Excel 汇总 → + export_workbook.py
│   └─ 全要 → scripts/run_pipeline.py <项目目录>
│
└─ 交付前 → validate_delivery.py --strict
```

## 权重方法选择

| 情况 | 方法 |
|------|------|
| 默认 | `entropy` |
| 指标波动大、相关性强 | `critic` |
| 用户要求等权或指标同等重要 | `equal` |
| 有专家成对比较 | AHP（手写，见 assets/04） |

## 一句话黄金路径

```bash
python scripts/run_pipeline.py my_project
```

## 禁止事项

- 不读原始文件就下结论
- 静默补值不写说明
- 图表数量 > 子问题数 + 2
- 跳过 validate_delivery 就声称「已完成」
